## Controlling rules

1. **Copyright ownership remains with Mark Charles McLaughlin unless a signed written assignment states otherwise.**
2. **“Public”, “public-facing”, “public canon”, and “public working copy” describe visibility and research status. They do not by themselves grant a copyright licence.**
3. **A DOI is a provenance and citation identifier, not a licence.** A module’s reference to the MKUFT DOI does not import the licence of a different deposited work.
4. **Rights attach to the exact material and version being used.** A licence attached to a frozen Zenodo deposit applies to that deposited material. It does not automatically license later GitHub revisions, later modules, or separate publications.
5. **Historical grants remain effective for the exact material licensed.** Nothing in this repository purports to revoke a permission already granted under CC BY 4.0 or CC BY-NC-SA 4.0.
6. **Current-version reservation does not recapture historical CC material.** Where a current file contains material already released in an exact historical CC BY 4.0 deposit, that historical material remains available from that deposit under CC BY 4.0. New expression, arrangement, additions, revisions, and current-version compilation are not automatically covered by that older licence.
7. **Where no exact file, publication, or version states a licence, the default is:**

```text
Copyright © 2026 Mark Charles McLaughlin
All rights reserved
```

8. **No repository-level software licence is granted.** Publication of manuscripts, specifications, hypotheses, equations, or protocols does not grant a software, source-code, patent, trademark, certification, or commercial-implementation licence.
9. **Copyright boundary:** these notices govern copyright-protected expression and any other rights actually held. They do not claim copyright ownership of facts, abstract ideas, scientific principles, mathematical relationships, methods, functionality, or independently written code as such, and they do not create patent rights.

## Practical reading rule

For any reuse, identify the exact object first:

1. the exact file or frozen deposit;
2. its version or commit;
3. any licence stated inside that exact object;
4. its specific DOI record, if any;
5. this matrix and `RIGHTS_AND_LICENSE_NOTICE.md` for the current repository position.

When those sources conflict, the licence attached to the exact material being used controls. A general provenance reference cannot broaden that licence.

# Source Issues and Cleanup Log

Author: Mark McLaughlin

This log records wording and formatting issues identified while creating the cleaned GitHub working copy.

Original source files remain untouched. Corrections here apply only to the GitHub working copy unless Mark explicitly designates them as canon edits to the originals.

## 1. Layer Convention Drift

Some older material defined MKUFT layers as:

- S = Substrate
- I = Information
- P = Perception/Boundary
- O = Output

The current formal convention is:

- S = Substrate
- I = Information
- P = Physical
- O = Observer

Action taken:

- GitHub working documents use the current formal convention.
- Older P/O wording is treated as superseded historical shorthand.

## 2. P-Layer Consciousness Wording

Some source wording referred to `P-layer consciousness` or similar formulations.

Issue:

- Under the current convention, P = Physical, not consciousness/perception.
- Consciousness/observer coherence belongs under O, with physical expression under P.

Action taken:

- Reworded to observer coherence, observer-state modulation, or O-layer support where appropriate.

## 3. Converted Symbol Damage

Some PDF/Word conversions contained damaged symbols or placeholder boxes, including forms such as:

- `Prealized`
- broken square placeholders for Ω, I(E), or event sets
- equation linebreak damage

Action taken:

- Normalised `Prealized` to `P_realized`.
- Rendered broken symbols explicitly where the meaning was clear.
- Used plain-text/Markdown-safe equations for stability.

## 4. Addendum Integration

Multiple addendum versions existed.

Action taken:

- The revised full standalone addendum is treated as the active addendum.
- Earlier short addenda are treated as superseded unless they contain useful wording not present in the revised version.
- The GitHub version absorbs the clean node/trajectory/coherence-cost structure.

## 5. Support vs Primary Canon

OCQS, LUCY, ESRT/ESF, and Voynich material are powerful support modules, but should not blur the theory spine.

Action taken:

- They are placed after the core, math appendix, formal addendum, experiments, and falsification summary.
- They are framed as support/application layers.

## 6. Originals Preserved

The GitHub repository is a clarified working copy.

It does not overwrite, erase, or rewrite the original uploads or prior papers.

Future work should add:

- full source-document archive folder,
- original PDFs/DOCXs if desired,

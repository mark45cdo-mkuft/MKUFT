# Provenance, DOI, and Attribution

Author: Mark Charles McLaughlin

## Purpose

This file records the publication and attribution structure for the MKUFT repository.

The repository contains multiple related research nodes. These nodes are connected, but they should not be collapsed into one undifferentiated blob.

## Core Rule

Each published node keeps its own citation identity, DOI, authorship, and licence status.

MKUFT is the backbone framework.

LUCY is treated as part of the MKUFT backbone unless separately published or separately versioned later.

Voynich is a separate applied node linked to MKUFT through ESRT/ESF and procedural-system analysis.

## LUCY Placement

LUCY is currently treated as part of the MKUFT backbone.

It is not a separate external theory in this repository.

Its function is to describe local threshold, membrane, and field behaviour within the MKUFT stack:

```text
LUCY -> LUCY-1 -> LUCY-2
```

Meaning:

- LUCY = threshold,
- LUCY-1 = membrane,
- LUCY-2 = sustained field.

If LUCY is later published as its own node, that node should receive its own DOI and citation identity.

## Voynich Node

The Voynich work is a separate applied node.

It is related to MKUFT, but not identical to MKUFT.

Its role is to apply MKUFT/ESRT/ESF reasoning to the Voynich Manuscript as a procedural or executable system.

Current repo position:

```text
Voynich = applied procedural-engine node linked to MKUFT
```

If a separate Voynich Zenodo DOI is active, it should be recorded here in the next pass.

## ESRT / ESF Placement

ESRT and ESF are method layers.

They support MKUFT by clarifying how executable systems are recognised, modelled, and tested.

They do not replace the MKUFT physical theory.
